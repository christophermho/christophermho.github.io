Context
AMD is a semiconductor company that designs and produces computer processors, graphics cards, and related technologies‚ÄîRadeon Software is the device driver and utility software package for their Radeon graphics cards and advanced processing units. It includes features like game profile management, overclocking and undervolting, performance monitoring, recording and streaming, etc.
During my internship, I worked on the release of Radeon Software: Adrenalin (yes, they spelled it wrong) 2020 Edition, which I'll refer to as RSX for short. I focused on validating the design by creating and usability testing a high fidelity prototype. Other responsibilities during my internship included starting RSX's first design system and designing relevant screens and overlays when necessary.
Current State
Prior to the release of RSX, Radeon Software: Adrenalin was in its 2019 Edition. This version had a dated UI that had been used for years‚Äîusers were extremely familiar and comfortable with the current information architecture and navigation.
Future State
In the the upcoming RSX driver update, Radeon Software would replace Radeon Settings as a comprehensive suite that included drivers, settings, and additional features for Radeon graphics cards. Some of the new innovative features included Radeon Boost, Radeon Anti-Lag, Radeon Image Sharpening, Radeon Integer Display Scaling, AMD Link Streaming, and Integrated Web Browsing.
The biggest change Radeon Software brought was a completely revamped user interface. Compared to its predecessor, Radeon Software was designed to have a more modern and user-friendly interface‚Äîthe intention was to allow users to quickly access the latest software features, game stats, performance reports, and driver updates all from one convenient location, creating an immersive gaming experience.
When I was brought on board, I was tasked with validating the experience of RSX. Despite the revamped design being close to completion, it had barely been tested with actual users. Using the existing design, it was my job to create a high-fidelity prototype and conduct usability testing. While the future state was more modern and sleek, what mattered was whether it was actually intuitive or not. Would users have an easy time understanding and navigating the new UI, let alone be willing to spend the time learning and adopting it?
User Flows
Since RSX included several new features, it was necessary to design user flows for both new and existing features. It was also important to include a variety of tasks for users of different experience levels with gaming and Radeon Software. How would the experience differ for novice vs. experienced users? With these notes in mind, I designed the following user flows and created a high fidelity prototype accordingly ‚Äî complete with animations, transitions, overlays, and keyboard shortcuts.
    User Flows/Tasks
  1. Open Radeon Software
  2. Launch a game (Gears 5)
  3. Open Radeon overlay and sidebar
  4. Check software and hardware settings
  5. Enable/disable Radeon FreeSync
  6. Enable and/or modify Radeon Boost
  7. Enable Radeon performance metrics overlay
  8. Open performance tools and overclock GPU
  9. Open performance tuning, modify fan speed
  10. Open web browser
  11. Check for an update
  12. Change default settings profile
  13. Open and change video settings
  14. Create a new scene for streaming
  15. Start a streaming session
  16. Enable and capture an instant replay
  17. Enable and record a video of desktop
  18. Edit and trim a video
  19. Upload and share a video
Usability Testing
After creating the prototype, it was time to test the new Radeon Software design. To properly synthesize the usability testing data, it was neccessary to know what types of users were being interviewed. To gain some context, I started by asking the users a couple of screening questions:
    Screening Questions
  • How experienced are you with Radeon Settings/Software?
  • Are you a Gamer, eSports Enthusiast, and/or Content Creator? (can check all or none)
  • How many hours do you spend gaming in a week? (Never, 1-10, 11-20, 21+)
Feedback from usability testing was mixed. Some users praised the new interface, additional features, and customization options. However, other users were more reluctant and much preferred the previous design. There were also concerns with the amount of RAM some of the new features would take up (i.e. bloatware). On the other hand, certain features like Radeon Boost, Anti-Lag, Image Sharpening, and Integer Display Scaling were highly anticipated by virtually all users.
The UI was smooth and visually appealing, but took up unnecessary space, required too much scrolling, and felt overly complicated and layered due to excessive nested menus
There was difficulty finding many features, especially ones that had been replaced or integrated with new features (ex. Frame Rate Target Control replaced by Radeon Chill)
Only the fullscreen hotkey was explained on launch, so users did not realize the existence of the sidebar view‚Äîled to the false belief that there was only a fullscreen overlay, which was disliked
Majority of users had little to no care for certain features (i.e. integrated web browser and the increased focus on recording and streaming) and questioned their necessity
Results
Due to product deadlines, RSX had to be released before changes could be properly implemented. Subsequent releases of Radeon Software have received updates based on feedback from both usability testing and actual use. Radeon Software is currently in its 2023 release and is used by millions of gamers around the globe. Below is an introductory video for the initial release of RSX.
Next Steps
RSX was released immediately after my internship ended‚Äîif I had more time, I would pay attention to different metrics and KPIs sourced from the live release. This would provide me with further data to gauge and measure where the new design succeeds and where it could be further improved.
Installation Success Rate‚Äîa high success rate would indicate a smooth onboarding process for users and willingness to adopt the new design
Feature Utilization‚Äîanalyzing which features are most commonly used could guide future design efforts by focusing on the most popular and valuable features
User Flow Metrics‚Äîmeasuring click-through rates and time spent on different tasks could help identify areas where the interface could be optimized for better usability
Lessons Learned
In the past, I worked in smaller teams on products with smaller scope, so working at a large company like AMD taught me valuable lessons. The larger scale of product meant more complexity and constant communication with the PM, lead designer, and other stakeholders. I also learned that complying with user anticipated deadlines can sometimes mean releasing a product without all the improvements we wanted to include. This can feel disheartening, but is a learning experience‚Äîeverything is not always smooth sailing in real work screnarios. Overall, the experience highlighted the importance of clear communication, teamwork, planning, and how larger companies operate.
