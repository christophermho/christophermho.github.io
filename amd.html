Context
Radeon Software is a device driver and utility software package for AMDâ€™s Radeon graphics cards and APUs. It includes features like game profile management, performance monitoring, recording/streaming, and more. During my internship, I worked on the release of Radeon Software: Adrenaline 2020 Edition (RSA 2020). I focused on improving the UX by designing and usability testing a high fidelity prototype.
Current State
Prior to the release of RSA 2020, Radeon Software was known as Radeon Settings (Adrenaline 2019 Edition). This version had a UI that had been used for years â€” users were extremely familiar with the current information architecture and navigation.
Future State
In the the upcoming RSA 2020 driver update, Radeon Software would have a completely redesigned UI and a number of new innovative features like Radeon Boost, Radeon Anti-Lag, Radeon Image Sharpening, Radeon Integer Display Scaling, AMD Link Streaming, and Integrated Web Browsing.
Designed to provide the user with a clean, modern and easy-to-use interface where they can quickly access the latest software features, game stats, performance reports, driver updates, and much more - all from one convenient location, making it even more convenient and easy to adjust their favorite features and get a fantastic gaming experience.
While I designed several screens for the RSA 2020 driver update, the majority of the UI was already complete when I began my internship. However, the UX of the new design had not been tested yet â€” using the mockups, it was my job to create a prototype and conduct usability testing. While the future state was more modern and sleek, what mattered was if it was actually intuitive or not. Would users have an easy time understanding and navigating the new UI, let alone opening it in the first place?
User Flows
Since RSA 2020 included several new features, it was necessary to design user flows for both new and existing features. It was also important to include a variety of tasks for users of different experience levels with gaming and Radeon Software. How would the experience differ for novice vs. experienced users? With these notes in mind, I designed the following user flows and created a high fidelity prototype accordingly â€” complete with animations and overlays.
    User Flows/Tasks
  1. Open Radeon Software
  2. Launch a game (Gears 5)
  3. Open Radeon overlay and sidebar
  4. Check software and hardware settings
  5. Enable/disable Radeon FreeSync
  6. Enable and/or modify Radeon Boost
  7. Enable Radeon performance metrics overlay
  8. Open performance tools and overclock GPU
  9. Open performance tuning, modify fan speed
  10. Open web browser
  11. Check for an update
  12. Change default settings profile
  13. Open and change video settings
  14. Create a new scene for streaming
  15. Start a streaming session
  16. Enable and capture an instant replay
  17. Enable and record a video of desktop
  18. Edit and trim a video
  19. Upload and share a video
Usability Testing
After creating the prototype, it was time to test the new Radeon Software design. To properly synthesize the usability testing data, it was neccessary to know what types of users were being interviewed. To gain some context, I started by asking the interviewees/users a couple of screening questions:
    Screening Questions
  • How experienced are you with Radeon Settings/Software?
  • Are you a Gamer, eSports Enthusiast, and/or Content Creator? (can check all or none)
  • How many hours do you spend gaming in a week? (Never, 1-10, 11-20, 21+)
    Key Insights
  • The UI was smooth and visually appealing, but took up an unnecessary amount of space and felt overcomplicated and layered with the exessive number of nested menus
  • The majority of users were willing to learn and get used to the new design, with a few reluctant users preferring the previous design
  • The new features (i.e. Radeon Boost/Anti-Lag/Image Sharpening/Integer Display Scaling) were highly anticipated by all users
  • There were some concerns with 'bloatware' using excessive RAM (ex. questioning the necessity of an integrated web browser)
Results
Due to product deadlines, RSA 2020 was released before changes could be implemented. However, subsequent releases of Radeon Software have received updates based on feedback from both usability testing and actual use.
